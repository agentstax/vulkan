The Rising Waterline

A message log is a tall glass that fills from the bottom. The waterline rises only as high as the solid water beneath it — it can never float over an air pocket. That single rule is the whole idea.

1 The picture in one breath

Offsets are stacked cells, 0 at the bottom, climbing upward as new messages arrive. Each cell is a pocket that either fills (the message settles) or stays full of air (still working). The waterline is the tallest level where everything below is solid.

⚠ Why this beats a naive single cursor. A plain cursor is one pointer that only moves to the next message if that exact message succeeded. The instant offset 1 dies, that pointer is frozen at 0 forever — head-of-line blocking, the whole group stalls behind one bad message. The waterline simply seals the dead cell and rises past it.

4 Two things, doing two jobs

Below the line: one integer

Everything submerged is fully settled (each message is done or dead). It doesn’t matter how it ended — it’s under water. The whole submerged region compresses to a single number: the waterline height. Old rows are garbage-collected. This part of the log never grows.

Above the line: the exception window

Above the surface lives the messy reality — some filled, some air, some sealed. These need real per-message state. The window’s size tracks the disorder (how far ahead workers race), not the size of the whole log. Quiet, in-order traffic → almost no window.

The roll-up: a lazy, background surge

Workers never touch the waterline. Each grabs a distinct cell (a short lease), fills it, moves on — no coordination, even across machines. Separately, a background roll-up checks: “are the cells directly above the line all settled now?” If so, it slides the line up to swallow the whole contiguous run at once. Filling one low pocket can let it cascade several levels in a single move.

Crash-safe by design: per-cell outcomes are durable; only un-saved roll-up progress can be lost. That’s harmless — a stale-low line just rises again on restart by re-checking the cells. The guarantee is at-least-once.

! Where this metaphor leaks

What it makes vivid

  • Contiguity is physical. Water cannot hover over air — so it’s obvious why the mark stops at the first gap.
  • The surge. Filling one low pocket visibly lets the line leap many levels — the cascade is intuitive.
  • Poison doesn’t block. A sealed cell that water rises past captures “dead counts as resolved” cleanly.
  • Two-tier storage. Submerged = one number; above = detailed state. The split reads naturally.

What it hides or distorts

  • Water is continuous; offsets are discrete. Real fills are independent per-cell events, not a smooth liquid surface.
  • Gravity implies order; processing has none. Workers fill cells in any order from any machine — nothing “falls” or settles by physics.
  • One waterline per group. Each consumer group has its own line over the same log; the single-glass image hides the per-group multiplicity.
  • The line isn’t a live sensor. It’s computed lazily in the background and can lag the true state; “a rising level” suggests instant, continuous tracking.
  • No real top. The log is unbounded and append-only; a glass implies a finite container that could overflow.
  • Dead isn’t “solid.” A sealed cell is retained elsewhere for audit, not actually merged into the clean submerged block — the metaphor makes it look the same as a fill.
The Rising Waterline — one mental model for the managed cursor. Best paired with the technical spec, not a substitute for it.
